Two important issues were found: a CSS fallback value mismatch in `.mobile-hud-expanded` that mispositions the expanded HUD on non-mobile viewports, and an unmemoized `visitedSet` in `CameraControls` that violates the project's hot-path object creation rule and causes stale visited-system checkmarks. ## Important Issues

### [I1] `.mobile-hud-expanded` uses wrong fallback `48px` for `--mobile-hud-height`

- **File:** `css/hud.css:1114, 1117`
- **Bug:** `.mobile-hud-expanded` uses `var(--mobile-hud-height, 48px)` as its fallback, while all 6 other usages of this variable use `0px` as fallback. On any viewport where `--mobile-hud-height` is not defined (i.e., desktop >600px), the expanded panel is pushed 48px below the top and its max-height reduced by 48px unnecessarily.
- **Impact:** Visual misposition of the mobile HUD expanded overlay if it were ever rendered outside the mobile media query context. Inconsistent fallback convention creates maintenance risk.
- **Suggested fix:** Change both `var(--mobile-hud-height, 48px)` to `var(--mobile-hud-height, 0px)` to match the established convention.
- **Confidence:** High
- **Found by:** Contract & Integration, verified

### [I2] `visitedSet` recreated on every render in CameraControls

- **File:** `src/features/navigation/CameraControls.jsx:36`
- **Bug:** `const visitedSet = new Set(game.getVisitedSystems())` is at render scope without `useMemo`. A new `Set` is allocated on every render (triggered by `LOCATION_CHANGED`, `PREFERENCES_CHANGED`, and `SHIP_NAME_CHANGED` events). The `sortedStars` useMemo at line 38 does not depend on `visitedSet`, so the visited checkmarks in the desktop star-finder dropdown may be stale after the first render.
- **Impact:** Violates CLAUDE.md rule "Never create objects in hot loops." Potential stale visited-system checkmarks in the desktop star finder. On mobile, the Set is allocated and immediately discarded (unused).
- **Suggested fix:** Wrap in `useMemo` with dependency on `currentSystemId` (from the `LOCATION_CHANGED` event hook already at line 26), and include it in the `sortedStars` memo or derive labels inline.
- **Confidence:** High
- **Found by:** Logic & Correctness, Concurrency & State, verified

- **[S4]** `css/variables.css:274` — `.system-info-panel` in consolidated block has no JSX counterpart. Dead CSS rule. Either add the class to the relevant component or remove from the selector. (Found by: Contract & Integration)
