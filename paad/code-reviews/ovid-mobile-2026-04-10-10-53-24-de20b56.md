## Important Issues

### [I1] Mobile system panel and MobileHUD can overlap
- **File:** `src/App.jsx:437-441`
- **Bug:** The `panelActive` prop excludes `showSystemPanel` on mobile (via `!isMobile && showSystemPanel`). The system panel renders fullscreen on mobile at `z-index: 20`, but the mobile HUD bar sits above at `z-index: 30`. Since `panelActive` is false when only the system panel is open, the HUD bar remains tappable and expandable (`z-index: 40`), creating a visual overlap where the expanded HUD covers the fullscreen system panel.
- **Impact:** Confusing UX — the user can expand the HUD over a fullscreen system panel, with two overlapping UI layers competing for attention. Users may not realize the system panel is still underneath.
- **Suggested fix:** Include `showSystemPanel` in `panelActive` on mobile too: `panelActive={activePanel !== null || viewMode === VIEW_MODES.ENCOUNTER || showSystemPanel}`, or add auto-collapse of the system panel when the HUD expands.
- **Confidence:** High
- **Found by:** Concurrency & State, Logic & Correctness

Using `data-mobile-hud` would be more consistent and resilient to CSS class renames. (Found by: Contract & Integration)
