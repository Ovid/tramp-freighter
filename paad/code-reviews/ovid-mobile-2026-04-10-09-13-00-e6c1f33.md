## Executive Summary

The mobile responsive design implementation is well-structured and follows the design spec closely. Four important issues were found: the MobileHUD can expand over encounter/system panels due to an incomplete dismiss callback, `MobileCameraToolbar` has two missing prop defaults that would crash on undefined, and the station interface uses the wrong z-index variable on mobile. ## Important Issues

### [I1] `onDismissPanel` cannot dismiss encounters or SystemPanel; MobileHUD expands over them
- **File:** `src/App.jsx:437-439`, `src/features/hud/MobileHUD.jsx:53-67`
- **Bug:** `panelActive` is `true` when `activePanel !== null`, `viewMode === ENCOUNTER`, or `showSystemPanel` (viewingSystemId !== null). But `onDismissPanel` only calls `handleClosePanel()`, which clears `activePanel` ﻗ it does NOT clear `viewingSystemId` or transition out of ENCOUNTER mode. When the user taps the MobileHUD bar while an encounter or system panel is open, `onDismissPanel()` is a no-op for those panel types, but `setExpanded(true)` still runs, causing the HUD drawer to expand **over** the encounter/system panel content.
- **Impact:** On mobile, a player in a pirate encounter can tap the HUD bar and get the full HUD overlay on top of encounter choices. The Dock/System Info buttons in the expanded HUD become accessible during encounters, which could cause state corruption.
- **Suggested fix:** Either (a) block expansion when `panelActive` is already true in the MobileHUD click handler (`if (willExpand && panelActive) return;`), or (b) expand `onDismissPanel` in App.jsx to also close the system panel and refuse during encounters.
- **Confidence:** High
- **Found by:** Contract & Integration, Concurrency & State

### [I2] `MobileCameraToolbar`: `stars.map()` crashes if `stars` prop is undefined
- **File:** `src/features/navigation/MobileCameraToolbar.jsx:46`
- **Bug:** `stars` has no default parameter value and no null guard. If the parent ever passes `undefined` (or omits the prop), `stars.map(...)` throws `TypeError`, crashing the toolbar and triggering the error boundary.
- **Impact:** Currently safe because the one caller (`CameraControls.jsx`) always provides a default `[]` from `sortedStars`. But the component has no self-defense ﻗ a future refactor could break it with no warning.
- **Suggested fix:** Add default parameter: `{ stars = [], ...rest }` in the destructuring.
- **Confidence:** High
- **Found by:** Error Handling & Edge Cases

### [I3] `MobileCameraToolbar`: `Object.entries(toggles)` crashes if `toggles` is undefined
- **File:** `src/features/navigation/MobileCameraToolbar.jsx:65`
- **Bug:** `Object.entries(toggles)` throws `TypeError` if `toggles` is `null` or `undefined`. No default prop is set. The crash only occurs when the user taps the Settings gear button (the popover is conditionally rendered), making it harder to catch in basic testing.
- **Impact:** Same as I2 ﻗ currently safe due to the caller, but fragile.
- **Suggested fix:** Destructure with default: `toggles = {}`.
- **Confidence:** High
- **Found by:** Error Handling & Edge Cases

### [I4] `#station-interface` mobile z-index uses `--z-camera-toolbar` (10) instead of panel-level z-index
- **File:** `css/hud.css` (mobile `#station-interface` block, ~line 728)
- **Bug:** On mobile, `#station-interface` is assigned `z-index: var(--z-camera-toolbar)` which is `10`. Every other full-screen mobile panel uses `var(--z-panel-fullscreen)` which is `20`. The station interface is the primary docked UI ﻗ if it renders at the same z-level as the camera toolbar, document order determines stacking. More importantly, if `--z-camera-toolbar` is ever adjusted for toolbar-specific reasons, the station menu's z-index changes as a side effect.
- **Impact:** Semantic mismatch creates fragile coupling between station menu and camera toolbar layering. Could cause station menu to render behind other panels in edge cases.
- **Suggested fix:** Use `z-index: var(--z-panel-fullscreen)` for `#station-interface` on mobile.
- **Confidence:** High
- **Found by:** Logic & Correctness
