## Suggestions

### [S1] Negative reputation label/ARIA inconsistency
- **File:** `src/features/dialogue/DialoguePanel.jsx:133`
- **Bug:** When `currentRep` is negative, the text label shows the raw value (e.g., "Trust: -10 / 100") but `aria-valuenow` (line 148) is clamped to 0 via `Math.max(0, ...)` and bar width (line 159) is clamped to 0%. Visual text and ARIA/bar disagree.
- **Impact:** Accessibility inconsistency for screen reader users. Low practical impact since Tanaka is the only quest NPC, starts at rep 0, and negative rep during her quest is uncommon.
- **Suggested fix:** Clamp the displayed text to match: `Trust: {Math.max(0, currentRep)} / 100`, or allow ARIA/bar to reflect negative values.
- **Confidence:** Medium
- **Found by:** Logic & Correctness, Error Handling & Edge Cases

### [S2] Stale JSDoc -- reputationTier documented as string, actually Object
- **File:** `src/game/game-dialogue.js:133`
- **Bug:** JSDoc says `reputationTier: string` but `getRepTier()` returns `{ name, min, max }`. DialoguePanel.jsx correctly accesses `.name` on it, proving the actual type is an object.
- **Impact:** Misleading documentation. A future developer trusting the JSDoc could access `reputationTier` as a string and get a runtime error.
- **Suggested fix:** Update JSDoc to `reputationTier: Object - { name: string, min: number, max: number }`.
- **Confidence:** High
- **Found by:** Contract & Integration
- **Note:** Pre-existing, not introduced by this diff.
