## Important Issues

### [I1] Condition bar black text invisible at low values — FIXED (d2406f5)
- **File:** `css/hud.css:144`
- **Bug:** `.condition-text` was changed from white to black for contrast against bright colored bars. However, the text is absolutely positioned at the center of the `.condition-bar-container` (which has background `var(--bg-overlay-darker)` = `rgba(0,0,0,0.5)` over the `#000` body). The colored bar only fills from the left based on the percentage. When a condition is low (e.g., fuel at 20%), the text at 50% sits over the dark unfilled area. Black text on near-black background is invisible. The white text-shadow (`rgba(255,255,255,0.3)`) is too faint to compensate.
- **Impact:** This is the inverse of the original S5 contrast problem. At critically low values -- precisely when the player most needs to read fuel/hull/engine/life-support percentages -- the text disappears. This is a functional regression.
- **Suggested fix:** Options: (1) Use white text with a stronger dark text-shadow (e.g., `text-shadow: 0 0 4px rgba(0,0,0,0.9), 1px 1px 2px rgba(0,0,0,0.8)`) which provides contrast on both light bars and dark backgrounds. (2) Add a semi-opaque background behind the text span to ensure consistent contrast. (3) Use a CSS mix-blend-mode approach.
- **Fix applied:** Option (1) — white text with strong dark text-shadow.
- **Confidence:** High
- **Found by:** Logic & Correctness specialist

### [I2] No disabled styling for `.tactical-option` in PirateEncounterPanel — FIXED (3fa5081)
- **File:** `css/panel/pirate-encounter.css` (missing rule), `src/features/danger/PirateEncounterPanel.jsx:246`
- **Bug:** The negotiate button uses `disabled={escalated}` with a `disabled` CSS class applied. However, `pirate-encounter.css` has no `.tactical-option.disabled` or `.tactical-option:disabled` CSS rule. The other encounter panels (inspection.css, mechanical-failure.css, distress-call.css) all have explicit disabled styling (`opacity: 0.5`, `cursor: not-allowed`, muted borders). The pirate encounter negotiate button when disabled looks identical to enabled options.
- **Impact:** After a failed negotiation, the player sees "Negotiate" looking fully clickable with no visual indication it's unavailable. The "They're done talking." text inside the button helps, but the button itself has no visual differentiation.
- **Suggested fix:** Add to `pirate-encounter.css`:
  ```css
  .tactical-option.disabled,
  .tactical-option:disabled {
    opacity: 0.5;
    cursor: not-allowed;
    border-color: rgba(255, 255, 255, 0.1);
  }
  .tactical-option.disabled:hover,
  .tactical-option:disabled:hover {
    border-color: rgba(255, 255, 255, 0.1);
    background-color: rgba(0, 0, 0, 0.3);
  }
  ```
- **Confidence:** High
- **Found by:** Logic & Correctness, Error Handling specialists (2/5)
